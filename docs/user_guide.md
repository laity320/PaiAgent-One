## 1. 环境准备

### 1.1 系统要求

| 组件 | 最低要求 |
|------|----------|
| 操作系统 | Windows 10+ / macOS 12+ / Linux |
| JDK | 21 或更高版本 |
| Node.js | 18.x 或更高版本 |
| MySQL | 8.0 或更高版本 |
| Maven | 3.8 或更高版本 |
| 内存 | 建议 4 GB 以上 |

### 1.2 大模型 API Key

使用 LLM 节点前，需要至少准备一个大模型厂商的 API Key：

| 厂商 | 获取方式 |
|------|----------|
| DeepSeek | 在 DeepSeek 开放平台注册并创建 API Key |
| 通义千问 | 在阿里云 DashScope 控制台创建 API Key |
| 智谱 AI | 在智谱 AI 开放平台注册并创建 API Key |
| OpenAI 兼容 | 任何兼容 OpenAI API 格式的服务 |

### 1.3 TTS 服务

工具节点（超拟人音频合成）需要配置对应的 TTS 服务 API。请根据使用的 TTS 服务商配置相关参数。

## 2. 安装部署

### 2.1 数据库初始化

1. 确保 MySQL 服务已启动

2. 执行数据库初始化脚本：

```bash
mysql -u root -p < backend/src/main/resources/db/migration/V1__init_schema.sql
```

该脚本会自动：
- 创建 `paiagent` 数据库
- 创建 4 张数据表（用户表、工作流表、执行记录表、大模型配置表）
- 插入默认管理员账号

> 默认管理员账号：`admin` / 密码：`admin123`

### 2.2 后端配置

编辑 `backend/src/main/resources/application.yml`，根据实际环境修改以下配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/paiagent?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的数据库密码

paiagent:
  jwt:
    secret: 替换为随机字符串（生产环境必须修改）
```

也可以通过环境变量设置：

```bash
# 数据库密码
set DB_PASSWORD=你的数据库密码

# JWT 密钥
set JWT_SECRET=你的JWT密钥
```

### 2.3 启动后端

```bash
cd backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动。

### 2.4 安装前端依赖

```bash
cd frontend
npm install
```

> 如果遇到权限问题，可使用：`npm install --cache "%USERPROFILE%\.npm_cache"`

### 2.5 启动前端

```bash
cd frontend
npm run dev
```

前端开发服务器将在 `http://localhost:3000` 启动。

### 2.6 访问应用

浏览器打开 `http://localhost:3000`，即可进入 PaiAgent-One 应用。

## 3. 首次使用

### 3.1 登录

1. 打开应用，自动跳转到登录页面
2. 使用默认管理员账号登录：
   - 用户名：`admin`
   - 密码：`admin123`
3. 登录成功后跳转到工作流列表页面

### 3.2 注册新用户

1. 在登录页面，切换到"注册"标签页
2. 填写用户名（2-64 字符）和密码（6 位以上）
3. 点击"注册"按钮
4. 注册成功后自动登录

## 4. 工作流编辑器

工作流编辑器是 PaiAgent-One 的核心功能，由以下区域组成：

```
┌──────────────────────────────────────────────────────┐
│                    顶部工具栏                          │
│  [品牌名] [工作流名称输入框] [新建] [加载] [保存] [调试]  │
├──────┬───────────────────────────────┬───────────────┤
│      │                               │               │
│ 节点  │          画布区域              │  节点配置      │
│ 库   │     (拖拽 + 连线)              │   面板        │
│ 面板  │                               │               │
│      │                               │               │
├──────┴───────────────────────────────┴───────────────┤
│              调试抽屉（展开时显示）                      │
└──────────────────────────────────────────────────────┘
```

### 4.1 画布操作

| 操作 | 方式 |
|------|------|
| 平移画布 | 按住鼠标左键拖拽空白区域 |
| 缩放画布 | 鼠标滚轮滚动 |
| 选中节点 | 单击节点 |
| 取消选中 | 单击画布空白区域 |
| 删除节点 | 选中节点后按 Delete 键 |
| 连线 | 从节点输出端口拖拽到另一节点的输入端口 |

### 4.2 添加节点

**方式：从左侧面板拖拽**

1. 在左侧"节点库"面板中找到需要的节点
2. 按住节点卡片，拖拽到画布区域
3. 松开鼠标，节点即被添加到画布

### 4.3 连接节点

1. 将鼠标移到节点底部的**输出端口**（小圆点）
2. 按住鼠标左键，拖拽出一条连线
3. 将连线拖到目标节点顶部的**输入端口**
4. 松开鼠标完成连线

### 4.4 配置节点

1. 单击画布中的某个节点
2. 右侧自动打开"节点配置面板"
3. 根据节点类型填写配置（详见第 6 节）
4. 配置会实时保存到画布状态

## 5. 节点类型详解

### 5.1 输入节点 (Input)

| 属性 | 说明 |
|------|------|
| 颜色标识 | 蓝色 |
| 连接点 | 仅有输出端口 |
| 功能 | 接收用户输入的文本内容，作为工作流的起始节点 |

每个工作流必须有且仅有一个输入节点。用户在调试面板中输入的文本将从此节点开始流转。

### 5.2 大模型节点 (LLM)

| 属性 | 说明 |
|------|------|
| 颜色标识 | 紫色 |
| 连接点 | 输入端口 + 输出端口 |
| 功能 | 调用大模型 API 进行文本生成 |

可用的大模型节点：

| 名称 | 厂商 | 默认模型 |
|------|------|----------|
| DeepSeek | DeepSeek | deepseek-chat |
| 通义千问 | 阿里云 | qwen-turbo |
| AI Ping | OpenAI 兼容 | gpt-3.5-turbo |
| 智谱 | 智谱 AI | glm-4-flash |

### 5.3 工具节点 (Tool)

| 属性 | 说明 |
|------|------|
| 颜色标识 | 橙色 |
| 连接点 | 输入端口 + 输出端口 |
| 功能 | 执行特定工具操作 |

当前可用工具节点：

| 名称 | 类型 | 功能说明 |
|------|------|----------|
| 超拟人音频合成 | TTS | 将文本转换为语音音频文件 |

### 5.4 输出节点 (Output)

| 属性 | 说明 |
|------|------|
| 颜色标识 | 绿色 |
| 连接点 | 仅有输入端口 |
| 功能 | 收集并输出工作流最终结果 |

每个工作流必须有且仅有一个输出节点。输出节点会自动检测内容类型：如果结果包含音频 URL，将显示音频播放器。

## 6. 节点配置说明

### 6.1 输入节点配置

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 输入变量名 | 输入参数的变量名称 | 否（默认 text） |

### 6.2 大模型节点配置

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 模型厂商 | 选择 LLM 服务商（deepseek/qwen/openai/zhipu） | 是 |
| 模型名称 | 具体模型标识（如 deepseek-chat, qwen-turbo） | 是 |
| Temperature | 生成随机性控制，范围 0-2，值越大越随机 | 否（默认 0.7） |
| Max Tokens | 最大生成 token 数 | 否（默认 2048） |
| 系统提示词 | System Prompt，定义 AI 角色和行为 | 否 |
| 用户提示词模板 | User Prompt 模板，支持变量引用 | 是 |

**变量引用语法**：在用户提示词模板中使用 `{{节点ID.output}}` 引用上游节点的输出内容。

示例：

```
请将以下内容改写为播客风格的对话脚本：

{{input_1.output}}
```

### 6.3 工具节点配置（TTS）

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 音色选择 | TTS 发声人（如 xiaoxiao, xiaoyi 等） | 否（默认 xiaoxiao） |
| 输出格式 | 音频格式（mp3/wav） | 否（默认 mp3） |

### 6.4 输出节点配置

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 输出引用 | 选择上游节点作为输出来源 | 是 |
| 输出模板 | 自定义输出格式，支持变量引用 | 否 |

## 7. 工作流调试

### 7.1 打开调试面板

点击顶部工具栏的 **"调试"** 按钮，底部展开调试抽屉。

### 7.2 执行工作流

1. 在调试面板的文本输入框中输入测试内容
2. 点击 **"执行"** 按钮
3. 系统将通过 SSE（Server-Sent Events）实时推送执行进度

### 7.3 查看执行过程

执行过程中，调试面板会显示：

```
步骤 1: 输入节点        ✅ 完成
步骤 2: 通义千问        🔄 执行中...
步骤 3: 超拟人音频合成   ⏳ 等待中
步骤 4: 输出节点        ⏳ 等待中
```

每个步骤的状态说明：

| 状态 | 标识 | 说明 |
|------|------|------|
| 等待中 | 灰色 | 节点尚未开始执行 |
| 执行中 | 蓝色/加载动画 | 节点正在执行 |
| 完成 | 绿色 | 节点执行成功，可查看输出 |
| 失败 | 红色 | 节点执行出错，显示错误信息 |

### 7.4 查看输出结果

- **文本输出**：直接在调试面板底部显示文本内容
- **音频输出**：显示内置音频播放器，可直接播放生成的音频

### 7.5 典型工作流示例

**AI 播客生成工作流**：

```
输入节点 → 通义千问（改写为播客脚本） → 超拟人音频合成 → 输出节点
```

操作步骤：

1. 拖拽"输入节点"到画布
2. 拖拽"通义千问"节点到画布
3. 配置通义千问节点：
   - 系统提示词："你是一个专业的播客脚本作家，擅长将任何内容改写为生动有趣的播客对话脚本。"
   - 用户提示词模板："请将以下内容改写为一段3分钟左右的播客脚本：\n\n{{input_1.output}}"
4. 拖拽"超拟人音频合成"节点到画布
5. 拖拽"输出节点"到画布
6. 按顺序连线：输入 → 通义千问 → 超拟人音频合成 → 输出
7. 配置输出节点，选择引用"超拟人音频合成"的输出
8. 保存工作流
9. 打开调试面板，输入一段文字，点击执行
10. 等待执行完成，播放生成的音频

## 8. 工作流管理

### 8.1 创建新工作流

1. 点击顶部工具栏的 **"新建"** 按钮
2. 画布自动重置为默认状态（包含一个输入节点和一个输出节点）
3. 在工具栏的名称输入框中为工作流命名

### 8.2 保存工作流

1. 编辑完工作流后，点击 **"保存"** 按钮
2. 首次保存会创建新工作流；后续保存会更新现有工作流
3. 保存内容包括：工作流名称、所有节点及其配置、所有连线关系

### 8.3 加载已有工作流

**方式一：从列表页面进入**
1. 在工作流列表页面，找到目标工作流
2. 点击"编辑"进入编辑器

**方式二：在编辑器中加载**
1. 点击顶部工具栏的 **"加载"** 按钮
2. 在弹出的对话框中选择要加载的工作流

### 8.4 删除工作流

1. 在工作流列表页面，找到目标工作流
2. 点击"删除"按钮
3. 确认后工作流将被归档（软删除）

## 9. 常见问题

### Q1: 前端启动后页面空白

**可能原因**：后端服务未启动，API 请求失败。

**解决方法**：确保后端服务在 `http://localhost:8080` 正常运行。

### Q2: 登录失败提示"用户名或密码错误"

**可能原因**：数据库未初始化或默认用户未创建。

**解决方法**：重新执行数据库初始化脚本 `V1__init_schema.sql`。

### Q3: 大模型节点执行失败

**可能原因**：
1. 未配置对应厂商的 API Key
2. API Key 余额不足
3. 网络无法访问 LLM 服务商 API

**解决方法**：
1. 检查后端配置中对应厂商的 API Key 是否正确
2. 确认 API Key 有效且有余额
3. 检查网络连通性

### Q4: TTS 音频合成失败

**可能原因**：TTS 服务未配置或服务不可用。

**解决方法**：检查 TTS 服务的接入配置，确认服务端是否正常运行。

### Q5: 工作流执行超时

**可能原因**：默认超时时间为 120 秒，复杂工作流可能超时。

**解决方法**：修改 `application.yml` 中的超时配置：

```yaml
paiagent:
  engine:
    default-timeout-seconds: 300  # 增大超时时间
    node-timeout-seconds: 120
```

### Q6: npm install 遇到 EPERM 权限错误

**可能原因**：npm 缓存目录在系统保护路径下。

**解决方法**：

```bash
npm install --cache "%USERPROFILE%\.npm_cache"
```

### Q7: Maven 编译失败，提示连接 localhost:8081 被拒绝

**可能原因**：项目 Maven 配置中包含本地 Nexus 仓库地址，但该服务未启动。

**解决方法**：项目的 `pom.xml` 已配置了 Maven Central 和阿里云镜像作为备用仓库，确保网络能访问外部 Maven 仓库即可。

### Q8: 如何扩展新的大模型厂商？

在后端 `llm/adapter/` 目录下：

1. 创建新的适配器类，继承 `OpenAiAdapter`（如果兼容 OpenAI API 格式）
2. 覆写 `getProvider()`、`getBaseUrl()`、`getApiKey()` 方法
3. 添加 `@Component` 注解，Spring 会自动注册

### Q9: 如何扩展新的工具节点？

在后端 `tool/handler/` 目录下：

1. 创建新的处理器类，实现 `ToolHandler` 接口
2. 实现 `getType()` 和 `execute()` 方法
3. 添加 `@Component` 注解

在前端 `constants/nodeRegistry.ts` 中：

1. 在 `NODE_REGISTRY` 数组中添加新工具节点的定义
2. 在 `components/CustomNodes/` 中如有需要创建自定义节点组件
